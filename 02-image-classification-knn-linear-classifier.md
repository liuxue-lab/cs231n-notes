## 1. 图像分类与语义鸿沟

图像分类的任务是：

```text
输入一张图像 → 输出一个类别
```

例如，输入一张图片，输出“猫”“狗”或“汽车”。

但人和计算机看到的图像并不一样。

人看到的是一只猫，而计算机最开始看到的只是一组数字。

一张 RGB 彩色图像可以表示为：

```math
\mathbf{I}\in\mathbb{R}^{H\times W\times 3}
```

其中：

- $H$ 表示图像高度；
- $W$ 表示图像宽度；
- $3$ 表示 RGB 三个颜色通道；
- 像素值通常位于 $0\sim255$。

像素数据和“这是一只猫”之间的差距，被称为**语义鸿沟**。

### 我的疑问：彩色图像不是应该有三张矩阵吗？

是的。

一张 RGB 图片实际上由三张二维矩阵组成：

```math
\mathbf{R}\in\mathbb{R}^{H\times W}
```

```math
\mathbf{G}\in\mathbb{R}^{H\times W}
```

```math
\mathbf{B}\in\mathbb{R}^{H\times W}
```

三张矩阵在第三个维度上堆叠起来：

```math
\mathbf{I}=\mathrm{stack}(\mathbf{R},\mathbf{G},\mathbf{B})
```

这里是“堆叠”，不是普通的相加。

假设三张矩阵的大小都是：

```text
H × W
```

堆叠后得到的彩色图像大小就是：

```text
H × W × 3
```

课件中只显示一张数字矩阵，通常是为了简化说明，只展示一个通道或一张灰度图。

### 我的疑问：矩阵中的方括号怎么理解？

例如：

```python
[
    [105, 112, 108],
    [ 91,  98, 102],
    [ 76,  85,  90]
]
```

最外层的方括号表示整个矩阵，每一组内层方括号表示一行数据。

如果是完整的 RGB 图像，每个像素中还有三个数：

```python
[
    [[105, 112, 108], [111, 104,  99]],
    [[ 91,  98, 102], [106, 104,  79]]
]
```

其中最里面的：

```text
[R, G, B]
```

才表示一个像素的三个颜色值。

上面的数据表示一张大小为：

```text
2 × 2 × 3
```

的彩色图像。

## 2. 图像分类为什么困难？

同一种物体在不同图像中的像素可能差别很大，常见原因包括：

- 观察角度变化；
- 光照变化；
- 背景变化；
- 物体遮挡；
- 形状变化；
- 尺度变化；
- 同一类别内部差异。

例如，同一只猫只平移几个像素，人仍然认为它是猫，但图像中的大量像素值已经发生改变。

另一方面，不同类别的图像也可能具有相似的颜色和纹理。

因此，很难直接人工编写一套固定规则识别所有猫，需要让模型从数据中自动学习分类规则。

## 3. 数据驱动方法

数据驱动方法的基本流程是：

1. 收集图像和对应标签；
2. 使用训练数据学习分类规则；
3. 使用新图像测试分类器。

可以概括为：

```text
收集带标签的数据
        ↓
训练分类器
        ↓
输入新图像
        ↓
输出预测类别
```

数据驱动方法与传统人工规则方法的主要区别是：

```text
人工规则方法：人编写识别规则

数据驱动方法：模型从训练数据中学习规则
```

KNN 是最简单的数据驱动分类方法之一。

## 4. 最近邻分类器

最近邻分类器会保存所有训练图像。

对于一张新的测试图像，它会：

1. 计算测试图像与所有训练图像的距离；
2. 找到距离最小的训练图像；
3. 使用这张训练图像的标签作为预测结果。

假设训练集为：

```math
\{(\mathbf{x}_i,y_i)\}_{i=1}^{N}
```

对于测试样本 $\mathbf{x}$，首先寻找距离最近的训练样本：

```math
i^*=\arg\min_i d(\mathbf{x},\mathbf{x}_i)
```

然后将其标签作为预测结果：

```math
\hat{y}=y_{i^*}
```

其中：

- $\mathbf{x}$ 是测试图像；
- $\mathbf{x}_i$ 是第 $i$ 个训练图像；
- $d(\mathbf{x},\mathbf{x}_i)$ 表示两张图像之间的距离；
- $y_i$ 是第 $i$ 个训练图像的标签。

这种方法几乎没有训练过程，但预测时需要和所有训练样本逐一比较。

因此它的特点是：

> 训练快，预测慢。

## 5. 图像之间的距离

图像可以展开成一个长向量，然后计算两个向量之间的距离。

例如，一张 CIFAR-10 图像的大小为：

```math
32\times32\times3
```

将其展开后，共有：

```math
32\times32\times3=3072
```

个数。

因此，一张 CIFAR-10 图像可以表示为一个 $3072$ 维向量。

### 5.1 L1 距离

L1 距离又称曼哈顿距离：

```math
d_{L1}(\mathbf{x},\mathbf{y})
=
\sum_{j=1}^{D}|x_j-y_j|
```

计算过程是：

> 对应元素相减，取绝对值，再将所有绝对值相加。

例如：

```math
\mathbf{x}=
\begin{bmatrix}
1\\
2
\end{bmatrix},
\qquad
\mathbf{y}=
\begin{bmatrix}
4\\
6
\end{bmatrix}
```

则：

```math
d_{L1}(\mathbf{x},\mathbf{y})
=
|1-4|+|2-6|
=
3+4
=
7
```

### 5.2 L2 距离

L2 距离又称欧氏距离：

```math
d_{L2}(\mathbf{x},\mathbf{y})
=
\sqrt{
\sum_{j=1}^{D}(x_j-y_j)^2
}
```

计算过程是：

> 对应元素相减、平方、求和，最后开平方。

对于刚才的数据：

```math
d_{L2}(\mathbf{x},\mathbf{y})
=
\sqrt{(1-4)^2+(2-6)^2}
```

```math
d_{L2}(\mathbf{x},\mathbf{y})
=
\sqrt{9+16}
=
5
```

L2 距离可以理解为高维空间中两个点之间的直线距离。

### 我的疑问：为什么菱形或圆上的点到原点距离都一样？

这里必须使用对应的距离公式。

对于二维点 $(x,y)$，它到原点的 L1 距离为：

```math
|x|+|y|
```

如果要求 L1 距离固定为 1，就有：

```math
|x|+|y|=1
```

所有满足这个条件的点，到原点的 L1 距离都等于 1，所以形成菱形。

例如：

```math
(1,0)
```

和：

```math
\left(\frac{1}{2},\frac{1}{2}\right)
```

都有：

```math
|x|+|y|=1
```

对于 L2 距离，如果要求距离固定为 1，就有：

```math
\sqrt{x^2+y^2}=1
```

两边平方后得到：

```math
x^2+y^2=1
```

这正是单位圆的方程，所以形成圆形。

可以简单记成：

```text
固定 L1 距离 → 菱形

固定 L2 距离 → 圆形
```

## 6. K 近邻算法 KNN

最近邻分类器只参考一张训练图像，因此容易受到异常样本和噪声影响。

KNN 会参考距离测试样本最近的 $K$ 个训练样本。

基本过程是：

1. 计算测试样本与所有训练样本的距离；
2. 找出距离最近的 $K$ 个样本；
3. 对这 $K$ 个样本的类别进行投票；
4. 将票数最多的类别作为预测结果。

假设最近的 $K$ 个样本组成集合 $\mathcal{N}_K(\mathbf{x})$，则 KNN 的预测结果可以表示为：

```math
\hat{y}
=
\arg\max_c
\sum_{i\in\mathcal{N}_K(\mathbf{x})}
\mathbf{1}(y_i=c)
```

其中：

- $c$ 表示某个类别；
- $\mathbf{1}(y_i=c)$ 是指示函数；
- 当 $y_i=c$ 时，指示函数等于 1；
- 否则等于 0。

例如，当 $K=5$ 时，最近的五个样本是：

```text
猫、猫、狗、猫、狗
```

投票结果为：

```text
猫：3 票
狗：2 票
```

因此预测结果为猫。

KNN 可以概括为：

```text
新样本
   ↓
寻找最近的 K 个训练样本
   ↓
进行类别投票
   ↓
输出票数最多的类别
```

### 我的理解：KNN 到底做了什么？

KNN 并没有真正学习“猫有哪些特征”。

它只是：

1. 记住所有训练图片；
2. 将新图片与训练图片逐一比较；
3. 找出最相似的几张图片；
4. 根据这些图片的标签进行投票。

因此，KNN 更接近一种基于记忆和相似度比较的方法。

### K 值的影响

当 $K$ 太小时：

- 容易受到噪声影响；
- 决策边界不稳定；
- 容易过拟合。

当 $K$ 太大时：

- 容易忽略局部结构；
- 预测结果可能被样本数量较多的类别控制；
- 容易欠拟合。

例如：

```text
K = 1：只看最近的一个样本，容易受异常点影响。

K = 5：综合附近五个样本，结果通常更加稳定。

K 很大：附近和较远的样本都会参与投票，局部特征可能被忽略。
```

$K$ 和距离函数都属于**超参数**，需要通过验证集选择。

## 7. 训练集、验证集和测试集

数据通常分成三个部分。

### 7.1 训练集

训练集用于学习模型参数。

例如在线性分类器中，模型会使用训练集学习权重矩阵 $W$ 和偏置向量 $\mathbf{b}$。

### 7.2 验证集

验证集用于选择超参数，例如：

- KNN 中的 $K$；
- 使用 L1 距离还是 L2 距离；
- 学习率；
- 正则化强度；
- 网络层数。

### 7.3 测试集

在模型结构、模型参数和超参数全部确定后，使用测试集评价模型的最终效果。

不能根据测试集结果反复修改模型，否则测试集就失去了客观评价作用。

正确流程是：

```text
训练集：学习模型参数

验证集：选择超参数

测试集：进行最终评价
```

## 8. KNN 为什么不适合直接用于图像分类？

KNN 的主要问题包括：

- 预测速度慢；
- 需要保存全部训练数据；
- 原始像素距离不能很好表达语义相似性；
- 对图像平移、旋转和光照变化比较敏感；
- 在高维空间中，距离的区分能力可能下降。

例如，同一张图片只平移一个像素，人仍然认为内容没有变化，但像素距离可能明显增大。

所以：

> 像素距离近，不一定代表语义相似；像素距离远，也不一定代表类别不同。

现代计算机视觉通常先使用神经网络提取图像特征，再在特征空间中进行分类，而不是直接比较原始像素。

```text
原始图像
   ↓
特征提取器
   ↓
高层语义特征
   ↓
分类器
```

## 9. 线性分类器

线性分类器属于参数化方法。

基本形式为：

```math
\mathbf{s}
=
f(\mathbf{x};W,\mathbf{b})
=
W\mathbf{x}+\mathbf{b}
```

其中：

- $\mathbf{x}$ 是输入图像向量；
- $W$ 是权重矩阵；
- $\mathbf{b}$ 是偏置向量；
- $\mathbf{s}$ 是所有类别的分数。

以 CIFAR-10 为例：

- 图像展开后维度为 $3072$；
- 共有 $10$ 个类别。

因此：

```math
\mathbf{x}\in\mathbb{R}^{3072}
```

```math
W\in\mathbb{R}^{10\times3072}
```

```math
\mathbf{b}\in\mathbb{R}^{10}
```

最终得到 10 个类别分数：

```math
\mathbf{s}
=
\begin{bmatrix}
s_1\\
s_2\\
\vdots\\
s_{10}
\end{bmatrix}
```

其中每一个 $s_i$ 对应一个类别的分数。

预测类别通常取分数最大的类别：

```math
\hat{y}=\arg\max_j s_j
```

### 我的疑问：参数 $W$ 是怎么来的？

$W$ 最开始通常是一组较小的随机数，例如：

```python
W = 0.001 * np.random.randn(10, 3072)
```

开始时，模型还没有学习任何规律，因此预测结果基本是随机的。

之后模型会不断执行：

```text
输入训练数据
    ↓
计算类别分数
    ↓
计算损失
    ↓
计算梯度
    ↓
更新 W 和 b
    ↓
重复训练
```

参数更新可以表示为：

```math
W
\leftarrow
W-\eta\frac{\partial L}{\partial W}
```

```math
\mathbf{b}
\leftarrow
\mathbf{b}
-\eta\frac{\partial L}{\partial\mathbf{b}}
```

最终的 $W$ 是通过训练数据学习得到的，不是人工提前计算好的。

$W$ 的每一行对应一个类别，可以粗略理解为这个类别的一套判断标准。

## 10. 线性分类器的三种理解

### 10.1 代数角度

线性分类器本质上是一次矩阵乘法：

```math
\mathbf{s}=W\mathbf{x}+\mathbf{b}
```

输入一个图像向量，输出所有类别的分数。

### 10.2 视觉角度

$W$ 的每一行都包含 $3072$ 个数。

将某一行恢复成：

```math
32\times32\times3
```

的图像形状后，可以将其看作某个类别的模板。

输入图像与某个类别模板越匹配，对应类别的分数通常越高。

例如，汽车类别的模板可能包含：

- 水平边缘；
- 车身轮廓；
- 轮胎附近的深色区域；
- 道路背景的颜色特征。

不过，这只是一种直观理解，并不代表权重矩阵真的保存了一张标准汽车图片。

### 10.3 几何角度

对于两个类别 $i$ 和 $j$，分类边界满足：

```math
\mathbf{w}_i^T\mathbf{x}+b_i
=
\mathbf{w}_j^T\mathbf{x}+b_j
```

整理后得到：

```math
(\mathbf{w}_i-\mathbf{w}_j)^T\mathbf{x}
+
(b_i-b_j)
=
0
```

这是一个超平面。

因此，线性分类器使用超平面划分不同类别。

它只能学习线性决策边界，无法直接表示特别复杂的分类规则。

## 11. 为什么需要损失函数？

给定一组参数 $W$ 和 $\mathbf{b}$，线性分类器可以输出类别分数。

但是只看分数，还不能直接评价当前参数是否合适。

损失函数会把模型的预测结果转换成一个数：

```text
预测越准确 → 损失越小

预测越错误 → 损失越大
```

损失函数可以写成：

```math
L(W,\mathbf{b})
```

训练目标是找到使损失最小的参数：

```math
W^*,\mathbf{b}^*
=
\arg\min_{W,\mathbf{b}}
L(W,\mathbf{b})
```

因此：

```text
损失函数负责评价模型参数好不好。

优化算法负责寻找更好的模型参数。
```

## 12. Softmax 分类器

线性分类器输出的原始分数称为 logits：

```math
\mathbf{s}
=
\begin{bmatrix}
s_1\\
s_2\\
\vdots\\
s_C
\end{bmatrix}
```

这些数可能为负，也不满足总和为 1，因此不能直接看作概率。

Softmax 将类别分数转换成概率：

```math
p_j
=
\frac{e^{s_j}}
{\sum_{k=1}^{C}e^{s_k}}
```

其中：

- $s_j$ 是第 $j$ 个类别的分数；
- $C$ 是类别总数；
- $p_j$ 是第 $j$ 个类别的预测概率。

转换后的概率满足：

```math
0<p_j<1
```

```math
\sum_{j=1}^{C}p_j=1
```

例如：

```text
猫：0.70
狗：0.20
汽车：0.10
```

表示模型认为这张图最可能属于猫。

为了防止指数运算数值溢出，实际计算时通常先减去最大分数：

```math
p_j
=
\frac{e^{s_j-\max(\mathbf{s})}}
{\sum_{k=1}^{C}e^{s_k-\max(\mathbf{s})}}
```

这种变换不会改变最终的 Softmax 概率。

## 13. 交叉熵损失

假设真实类别是猫，真实标签使用 one-hot 编码表示为：

```math
\mathbf{y}
=
\begin{bmatrix}
1\\
0\\
0
\end{bmatrix}
```

模型预测概率为：

```math
\mathbf{p}
=
\begin{bmatrix}
0.7\\
0.2\\
0.1
\end{bmatrix}
```

交叉熵损失为：

```math
L
=
-\sum_{j=1}^{C}y_j\log p_j
```

由于 one-hot 标签中只有正确类别对应的位置为 1，其余位置都是 0，所以：

```math
L=-\log p_y
```

其中，$p_y$ 是模型分配给真实类别的概率。

在这个例子中：

```math
L=-\log(0.7)\approx0.357
```

### 我的疑问：这个损失函数怎么理解？

只需要记住：

> 模型给正确类别的概率越高，损失越小。

| 正确类别概率 $p_y$ | 交叉熵损失 $-\log p_y$ |
|---:|---:|
| 0.99 | 0.010 |
| 0.90 | 0.105 |
| 0.50 | 0.693 |
| 0.10 | 2.303 |
| 0.01 | 4.605 |

当模型非常自信地预测正确时：

```math
p_y\rightarrow1
```

损失接近 0。

当模型非常自信地预测错误时：

```math
p_y\rightarrow0
```

损失会变得非常大。

因此，损失函数可以理解为：

> 给模型当前预测错误的严重程度打分。

## 14. 最大似然估计

最大似然估计希望选择一组参数 $\theta$，使训练数据中的真实标签最容易出现。

对于训练集：

```math
\{(\mathbf{x}_i,y_i)\}_{i=1}^{N}
```

似然函数可以写成：

```math
\mathcal{L}(\theta)
=
\prod_{i=1}^{N}
p_{\theta}(y_i\mid\mathbf{x}_i)
```

最大似然估计的目标是：

```math
\theta^*
=
\arg\max_{\theta}
\prod_{i=1}^{N}
p_{\theta}(y_i\mid\mathbf{x}_i)
```

通俗地说：

> 选择一组参数，让模型对训练数据的真实标签最不意外。

由于多个很小的概率相乘容易产生数值下溢，通常对似然取对数：

```math
\theta^*
=
\arg\max_{\theta}
\sum_{i=1}^{N}
\log p_{\theta}(y_i\mid\mathbf{x}_i)
```

再将最大化问题转换成最小化负对数似然：

```math
\theta^*
=
\arg\min_{\theta}
\left[
-\sum_{i=1}^{N}
\log p_{\theta}(y_i\mid\mathbf{x}_i)
\right]
```

这就是分类任务中的交叉熵损失。

因此：

```text
最大化真实类别的似然
        等价于
最小化负对数似然
        等价于
最小化交叉熵损失
```

## 15. 熵、交叉熵和 KL 散度

### 15.1 熵

熵衡量一个概率分布自身的不确定性：

```math
H(P)
=
-\sum_x P(x)\log P(x)
```

概率越平均，不确定性越大，熵越大。

概率越集中，不确定性越小，熵越小。

例如：

```text
[0.5, 0.5]：结果很不确定，熵较大。

[0.99, 0.01]：结果比较确定，熵较小。
```

### 15.2 交叉熵

交叉熵衡量使用预测分布 $Q$ 描述真实分布 $P$ 时的代价：

```math
H(P,Q)
=
-\sum_x P(x)\log Q(x)
```

在分类问题中：

- $P$ 是真实标签分布；
- $Q$ 是模型预测的概率分布。

预测分布越接近真实分布，交叉熵越小。

### 15.3 KL 散度

KL 散度衡量真实分布 $P$ 和预测分布 $Q$ 之间的差异：

```math
D_{\mathrm{KL}}(P\|Q)
=
\sum_x
P(x)
\log
\frac{P(x)}{Q(x)}
```

KL 散度满足：

```math
D_{\mathrm{KL}}(P\|Q)\geq0
```

当且仅当两个分布相同时：

```math
D_{\mathrm{KL}}(P\|Q)=0
```

### 15.4 三者的关系

三者之间满足：

```math
H(P,Q)
=
H(P)
+
D_{\mathrm{KL}}(P\|Q)
```

训练数据的真实分布 $P$ 是固定的，所以 $H(P)$ 是常数。

因此，最小化交叉熵：

```math
\min H(P,Q)
```

等价于最小化 KL 散度：

```math
\min D_{\mathrm{KL}}(P\|Q)
```

也就是让预测分布 $Q$ 逐渐接近真实分布 $P$。

可以简单记成：

```text
熵：真实分布本身有多不确定

交叉熵：使用预测分布描述真实分布的代价

KL 散度：预测分布与真实分布相差多少
```

## 16. 整个训练集的损失

如果训练集中有 $N$ 个样本，每个样本的损失为 $L_i$，平均损失为：

```math
L
=
\frac{1}{N}
\sum_{i=1}^{N}L_i
```

对于 Softmax 分类器，第 $i$ 个样本的损失为：

```math
L_i
=
-\log
\left(
\frac{e^{s_{y_i}}}
{\sum_{j=1}^{C}e^{s_j}}
\right)
```

因此，整个训练集的平均 Softmax 损失为：

```math
L
=
-\frac{1}{N}
\sum_{i=1}^{N}
\log
\left(
\frac{e^{s_{y_i}}}
{\sum_{j=1}^{C}e^{s_j}}
\right)
```

模型训练就是不断调整 $W$ 和 $\mathbf{b}$，使平均损失下降。

如果共有 $C$ 个类别，随机初始化时，每个类别的预测概率通常接近：

```math
p_j\approx\frac{1}{C}
```

所以初始交叉熵损失约为：

```math
L\approx-\log\left(\frac{1}{C}\right)=\log C
```

例如，CIFAR-10 有 10 个类别：

```math
L\approx\log10\approx2.303
```

因此，如果一个未经训练的 CIFAR-10 Softmax 分类器初始损失约为 2.3，通常是正常现象。

## 17. 多分类 SVM 损失

多分类 SVM 损失又称 Hinge Loss。

对于第 $i$ 个样本，其损失为：

```math
L_i
=
\sum_{j\neq y_i}
\max
\left(
0,
s_j-s_{y_i}+\Delta
\right)
```

其中：

- $s_{y_i}$ 是正确类别的分数；
- $s_j$ 是第 $j$ 个错误类别的分数；
- $\Delta$ 是安全间隔，通常取 1。

SVM 要求：

```math
s_{y_i}\geq s_j+\Delta
```

也就是说：

> 正确类别的分数不仅要最高，还要至少比错误类别高出一定距离。

例如：

```text
正确类别分数：5.0
错误类别分数：3.0
安全间隔：1.0
```

此时：

```math
3.0-5.0+1.0=-1.0
```

取最大值后：

```math
\max(0,-1.0)=0
```

说明已经满足安全间隔，不产生损失。

如果：

```text
正确类别分数：3.0
错误类别分数：2.8
安全间隔：1.0
```

则：

```math
2.8-3.0+1.0=0.8
```

因此会产生 $0.8$ 的损失。

Softmax 和 SVM 的区别可以简单理解为：

```text
Softmax：关注模型给各类别分配的概率。

SVM：关注正确类别分数是否比错误类别高出足够距离。
```

## 18. 第二节课总结

这一节需要重点记住：

1. 图像在计算机中是 $H\times W\times3$ 的数字张量。
2. RGB 图像由 R、G、B 三张二维矩阵沿第三个维度堆叠而成。
3. 像素数据和物体语义之间存在语义鸿沟。
4. KNN 根据距离最近的 $K$ 个训练样本投票分类。
5. L1 距离是对应元素绝对差之和。
6. L2 距离是欧氏距离。
7. $K$ 和距离函数属于超参数，需要通过验证集选择。
8. 原始像素距离不能很好地表达图像语义。
9. 线性分类器使用 $\mathbf{s}=W\mathbf{x}+\mathbf{b}$ 计算类别分数。
10. 参数 $W$ 从较小的随机值开始，通过降低损失逐渐学习得到。
11. Softmax 将类别分数转换为概率。
12. 交叉熵主要关注模型给正确类别分配的概率。
13. 正确类别的预测概率越高，交叉熵损失越小。
14. 最小化交叉熵等价于最大化正确标签的似然。
15. 多分类 SVM 要求正确类别分数比错误类别至少高出一个安全间隔。
16. 损失函数负责评价模型，优化算法负责更新参数。

整节课的主线可以概括为：

```text
图像像素
    ↓
定义图像之间的距离
    ↓
最近邻与 KNN 分类
    ↓
发现原始像素距离的局限
    ↓
引入参数化线性分类器
    ↓
计算类别分数
    ↓
使用 Softmax 或 SVM 定义损失
    ↓
通过优化算法更新 W 和 b
```
